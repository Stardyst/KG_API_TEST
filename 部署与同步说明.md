# 知识图谱本地测试接口部署与同步说明

## 一、代码仓库内容

Git 仓库只保存接口代码、前端页面、文档和依赖清单。

大数据文件不进入 Git，使用单独的数据包迁移。

## 二、新主机首次部署

1. 拉取代码：

```powershell
git clone <你的 GitHub 仓库地址>
cd <仓库目录>
```

2. 安装 Python 依赖：

```powershell
python --version
pip install -r kg_api/requirements.txt
```

3. 解压数据包到仓库根目录。

解压后必须存在：

```text
kg_source/exports/full_static_graph.json
kg_source/reports/event_field_index.json
```

4. 启动服务：

```powershell
python -m uvicorn kg_api.app:app --host 127.0.0.1 --port 8010
```

如果需要局域网其他机器访问：

```powershell
python -m uvicorn kg_api.app:app --host 0.0.0.0 --port 8010
```

5. 浏览器访问：

```text
http://127.0.0.1:8010
```

健康检查：

```text
http://127.0.0.1:8010/api/健康检查
```

## 三、更新同步

本机更新代码后：

```powershell
git add .
git commit -m "更新接口或页面"
git push
```

另一台主机同步代码：

```powershell
git pull
```

然后重启服务。

如果数据也更新，需要重新复制最新的数据包，并覆盖解压到仓库根目录。

## 四、重启服务

如果服务是在前台窗口运行，按 `Ctrl + C` 停止，然后重新执行：

```powershell
python -m uvicorn kg_api.app:app --host 127.0.0.1 --port 8010
```

如果服务用于局域网访问：

```powershell
python -m uvicorn kg_api.app:app --host 0.0.0.0 --port 8010
```

## 五、不需要迁移 Neo4j

当前接口运行不依赖 Neo4j 服务，只读取导出的静态图谱文件。

Neo4j 只在需要重新导入、重新导出、重新生成图谱数据源时才需要部署。
