# 知识图谱本地测试接口文档

## 1. 接口目标

本接口用于项目方通过网页访问本地 25 类事件知识图谱数据。项目方可选择事件类型，查看该事件类型支持的筛选字段，并根据筛选条件查询对应知识图谱结果。

接口返回数据格式统一为：

```json
{
  "staticNodes": [],
  "staticLinks": []
}
```

当前数据源来自本地导出的完整图谱文件：

```text
kg_source/exports/full_static_graph.json
```

Neo4j 暂作为数据生产、导入和校验工具，不作为接口实时查询依赖。

## 2. 通用规则

请求方式统一为：

```text
POST
```

请求体统一使用中文字段。

所有接口返回 JSON。

字段匹配规则：

- `事件类型` 为必填字段。
- `筛选条件` 可为空。
- 筛选字段必须来自“可筛选字段接口”返回结果。
- 字符串筛选值采用包含匹配。
- 数组筛选值内部为“或”关系。
- 不同筛选字段之间为“且”关系。

示例：

```json
{
  "事件类型": "船舶走私",
  "筛选条件": {
    "位置": ["广东", "汕头"],
    "走私物品": "柴油"
  }
}
```

含义：

```text
事件类型 = 船舶走私
并且 位置 包含 广东 或 汕头
并且 走私物品 包含 柴油
```

## 3. 事件类型列表接口

接口路径：

```text
POST /api/事件类型列表
```

功能说明：

用于获取系统当前支持的全部事件类型。

请求体：

```json
{}
```

响应示例：

```json
{
  "事件类型列表": [
    "船舶碰撞事故",
    "船舶非法采砂",
    "船舶非法排污",
    "船舶非法倾废",
    "船舶走私"
  ]
}
```

## 4. 可筛选字段接口

接口路径：

```text
POST /api/可筛选字段
```

功能说明：

根据指定事件类型，返回该事件类型支持的筛选字段。筛选字段来源于本地实体类型统计。

请求体：

```json
{
  "事件类型": "船舶走私"
}
```

响应示例：

```json
{
  "事件类型": "船舶走私",
  "可筛选字段": [
    "涉案主体",
    "走私物品",
    "船名",
    "过程时间点",
    "查获机构",
    "抓获时间",
    "法律法规",
    "位置",
    "伪装手段",
    "货物特征",
    "工具设备"
  ]
}
```

## 5. 字段候选值接口

接口路径：

```text
POST /api/字段候选值
```

功能说明：

根据事件类型和字段名称，返回该字段在本地数据中的候选值及出现次数。

请求体：

```json
{
  "事件类型": "船舶走私",
  "字段": "走私物品"
}
```

响应示例：

```json
{
  "事件类型": "船舶走私",
  "字段": "走私物品",
  "候选值": [
    {
      "值": "柴油",
      "命中次数": 61
    },
    {
      "值": "成品油",
      "命中次数": 8
    }
  ]
}
```

## 6. 知识图谱查询接口

接口路径：

```text
POST /api/知识图谱查询
```

功能说明：

根据事件类型和筛选条件查询知识图谱，并返回符合项目方要求的 `staticNodes/staticLinks` 数据。

请求体示例：

```json
{
  "事件类型": "船舶走私",
  "筛选条件": {
    "位置": "广东",
    "走私物品": "柴油"
  }
}
```

无筛选条件示例：

```json
{
  "事件类型": "船舶走私",
  "筛选条件": {}
}
```

响应示例：

```json
{
  "事件类型": "船舶走私",
  "命中事件数": 61,
  "节点数": 320,
  "关系数": 410,
  "staticNodes": [
    {
      "id": "node1",
      "name": "柴油",
      "ontoid": "",
      "ontName": "走私物品",
      "description": ""
    }
  ],
  "staticLinks": [
    {
      "id": "link1",
      "source": "node1",
      "target": "node2",
      "label": "运输"
    }
  ]
}
```

## 7. 结果下载接口

接口路径：

```text
POST /api/知识图谱下载
```

功能说明：

根据查询条件生成 JSON 文件下载。请求体与“知识图谱查询接口”一致。

请求体：

```json
{
  "事件类型": "船舶走私",
  "筛选条件": {
    "走私物品": "柴油"
  }
}
```

## 8. 全量知识图谱下载接口

接口路径：

```text
GET /api/全量知识图谱下载
```

功能说明：

导出当前接口运行时加载的完整知识图谱 JSON。导出内容包含原始图谱数据和接口运行时补充的测试字段数据。

前端页面提供“全量导出”按钮，并根据响应 `Content-Length` 显示下载进度。

响应内容：

```json
{
  "事件类型数": 25,
  "节点数": 85268,
  "关系数": 111361,
  "staticNodes": [],
  "staticLinks": []
}
```

## 9. 黑名单图谱接口

接口路径：

```text
GET /api/黑名单图谱
```

功能说明：

基于现有违法案例中的船舶相关实体，抽取违法船舶黑名单图谱。

当前黑名单抽取规则：

- 船名来源字段包括 `船名`、`船舶` 及船舶相关实体字段。
- MMSI 从相关实体文本中按 9 位号码规则抽取。
- 输出关系结构为 `违法事件类型 -> 船名 -> MMSI`。
- 当 MMSI 缺失时，保留船名节点，MMSI 为空。

响应内容：

```json
{
  "结果类型": "黑名单图谱",
  "事件类型": "违法船舶黑名单",
  "命中事件数": 100,
  "节点数": 50,
  "关系数": 80,
  "黑名单条目": [],
  "staticNodes": [],
  "staticLinks": []
}
```

## 10. 黑名单下载接口

接口路径：

```text
GET /api/黑名单下载
```

功能说明：

导出黑名单图谱 JSON 文件。页面提供“导出黑名单”按钮。

## 11. 课题要求补充字段

为满足课题三任务书中关于实体类型覆盖的测试需要，接口运行时补充以下字段，使其可通过“可筛选字段”“字段候选值”“知识图谱查询”接口访问：

- 船东
- 船舶管理公司人员
- 船舶驾引人员
- 船上乘客
- 工程船
- 交通流量
- 交通流分布
- 气象
- 水文
- 港口
- 航道
- 锚地
- 渔区
- 地方条例

该补充层用于接口测试，后续可由其他子课题交接的真实数据源替换。

响应形式：

```text
application/json 文件下载
```

建议文件名：

```text
船舶走私_查询结果.json
```

## 8. 错误响应格式

统一错误响应：

```json
{
  "错误": true,
  "错误类型": "参数错误",
  "错误信息": "事件类型不能为空"
}
```

常见错误类型：

```text
参数错误
事件类型不存在
筛选字段不存在
查询结果为空
服务器错误
```

示例：

```json
{
  "错误": true,
  "错误类型": "筛选字段不存在",
  "错误信息": "字段【走私物品】不属于事件类型【船舶碰撞事故】"
}
```

## 9. 页面功能对应关系

页面初始化：

```text
调用 /api/事件类型列表
```

选择事件类型：

```text
调用 /api/可筛选字段
```

选择筛选字段：

```text
调用 /api/字段候选值
```

点击查询：

```text
调用 /api/知识图谱查询
```

点击下载：

```text
调用 /api/知识图谱下载
```

## 10. 第一版实现范围

第一版实现以下功能：

```text
事件类型列表
可筛选字段
字段候选值
知识图谱查询
JSON 下载
本地测试网页
```

```text
在线修改数据
历史记录
多用户管理
数据上传
复杂图谱编辑
```

该版本目标是满足项目方本地联调和数据格式验证。
